# Samarth's Terminal Text Editor

A single-file terminal text editor written in C using POSIX APIs (raw mode, ANSI escape sequences, no external libraries). Based on the [Kilo tutorial](https://viewsourcecode.org/snaptoken/kilo/); extended with **snapshot undo/redo** and a **reproducible benchmark harness**.

## Features

- Raw terminal I/O (`termios`) with full-screen ANSI rendering
- Line-based buffer with tab expansion (8-space stops) and tab-aware cursor mapping
- Vertical and horizontal scrolling
- File open/save (Ctrl-S), save-as prompt, quit with unsaved warning (Ctrl-Q)
- Incremental search (Ctrl-F)
- Undo/redo (Ctrl-Z / Ctrl-Y) with snapshot stack (50 levels)
- Append-buffer render pipeline (single `write()` per frame)
- Non-interactive benchmark mode (`--benchmark`)

## Build and run

Requires Linux or WSL2 (uses `termios`, `unistd.h`, `getrusage`).

```bash
make main_c
./main_c                    # open empty buffer
./main_c myfile.txt         # open file
./main_c --benchmark file   # run benchmarks and exit
```

## Benchmarks

### Methodology

- **Environment:** WSL2, Linux 6.x, Intel i5-12450H (see `bench/results.txt` for full host info)
- **Test files:** Generated by `scripts/generate_test_files.sh` (~80 chars/line)
- **Command:** `./main_c --benchmark bench/<size>.txt`
- **Open time:** Wall-clock from `fopen` through last row loaded
- **Peak RSS:** `getrusage(RUSAGE_SELF).ru_maxrss` after load (KB on Linux)
- **Render time:** 100 iterations, first 3 discarded as warm-up; terminal fixed at 80x22 in non-TTY benchmark mode
  - **build:** append-buffer assembly only
  - **e2e:** build + `write()` to `/dev/null`
- **Search time:** Full-buffer linear scan with query `"aaaa"` via `strstr` on each row's render buffer

Reproduce all benchmarks:

```bash
make bench
```

Results are written to `bench/results.txt`.

### Results (measured 2026-06-21)

| File size | Lines | Open time | Peak RSS | Avg frame (build) | Avg frame (e2e) | Search time |
|-----------|------:|----------:|---------:|------------------:|------------------:|------------:|
| ~1 MB | 12,500 | 31 ms | 4.0 MB | 0.001 ms | 0.001 ms | 0.03 ms |
| ~10 MB | 125,000 | 277 ms | 28.0 MB | 0.001 ms | 0.001 ms | 0.37 ms |
| ~50 MB | 625,000 | 1.4 s | 135 MB | 0.001 ms | 0.001 ms | 1.8 ms |

**Practical limit:** Render speed stays constant (only visible rows are drawn). Open time and memory scale linearly with file size; ~50 MB / 625K lines loads successfully on this machine. Usability for editing depends on available RAM (entire file is held in memory).

## What makes this distinct from a tutorial project

1. **Undo/redo** — Snapshot-based history stack with deep row copies (not in base Kilo tutorial)
2. **Benchmark harness** — `--benchmark` mode plus scripts for reproducible open/render/search metrics
3. **Systems depth** — Raw mode, cx/rx tab mapping, append-buffer rendering, and scroll offsets are interview-explainable

This is **not** a novel architecture or production editor; it is an extended Kilo implementation with measurable performance characteristics.
